# Workspace Git Workflow Rules

- **Branching Strategy**:
  - Always perform development work and commits on the `dev` branch.
  - Push changes directly to the remote `dev` branch (`origin/dev`) whenever updates are completed.
  - **NEVER** push directly to the `main` branch.
  - **ALWAYS** prompt the user for explicit confirmation and approval before merging or pushing code changes to the `main` branch when work is done.
